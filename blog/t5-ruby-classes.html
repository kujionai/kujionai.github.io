Ruby Classes
Let's hang out.

So, think of an object. Any object. Did you think of a caper? Didn't think so. However, while the salted and pickled Capparis spinosa bud (that's a caper!) stands on its own as a glorious ingredient, why let it hang out with objects that serve a similar purpose?

We can group objects with a class, which is used to construct an object. Classes usually contain multiple methods, like this:

class NYC_Bagel
  def initialize(component, color)
    # Instance variables
    @component = component
    @color = color
  end

  def description
    puts "I am #{@component}! Check out my lovely #{@color} color."
  end

  def essential
    puts "I'm an essential component of the classic NYC bagel!"
  end
end

We've got the method descriptions that puts the description of a componenent of the classic NYC bagel, and the method essential that declares each ingredients' undeniable importance. Both methods are hanging out in our class NYC_Bagel.

Now, instead of creating new methods each time we want to reference a new NYC bagel component (capers inluded, of course), we can just refer to them as instances of the NYC_Bagel class like this:

d = NYC_Bagel.new("cream cheese", "off-white")
d.description
d.essential

d = NYC_Bagel.new("lox", "pink")
d.description
d.essential

d = NYC_Bagel.new("capers", "emerald")
d.description
d.essential

Sweet! We created new instances of our NYC_Bagel class and called the methods on each instance. Here's what we'll get when we run this in our terminal:

 I am cream cheese! Check out my lovely off-white color.
=> I'm an essential component of the classic NYC bagel!

=> I am lox! Check out my lovely pink color.
=> I'm an essential component of the classic NYC bagel!

=> I am capers! Check out my lovely emerald color.
=> I'm an essential component of the classic NYC bagel!

Awesome! That was so much easier with a class. Keep it tasty!
