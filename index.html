Git & Github
Git ready for some time travel.

I made some games, but I'm pretending like I didn't because they all turned out weird.
- Shigeru Miyamoto

Have you ever... wanted to go back to before you did something you really wish you maybe hadn’t done?

For example, I once ate seventeen delicious bagels in one 32-minute sitting. Maybe I wish I hadn’t eaten so many bagels.

If you’re a computer programmer, you can do just that - return to a previous version of reality in which something that was done is yet-to-be-done. They call this magical tool version control.

Here’s how it works.

Let’s say I use a handy version control management system called git.

With git, I can manage all my versions on my computer. Except (with inspiration from the the ingenious Stephanie Oh's awesome blog post Git vs. GitHub for Dummies), instead of managing my code, let's just say it manages my life.

I save versions of myself regularly. I saved the version of me that had eaten eleven bagels in one sitting. I saved the even older version of me that had eaten a mere six bagels in one sitting. Heck, I even saved the long-forgotten version of me, decades past, that hadn’t even heard of bagels!

So, back to the present me, seventeen bagels strong. And by strong, I mean very, very ill. I will spare you all details, because this blog post is about web development and not puke.

Whatever though, because I’ve got a FREAKING TIME MACHINE. I’m going back in time to before I even heard of bagels.

Heard of what? No idea. Heck yeah. Version control.

But, you know, there’s no reason to keep versions of your code, or versions of your life , to yourself. In my case, I’ve got to share that 17-bagel me with the world.

This is where Github comes in.

While git lets my code hang out locally on my machine, Github is a literal hub on the internet where everybody can share their stuff.

Makes a lot of sense to share your code. When you put it out there, others have the opportunity to learn from it.

With that in mind, you might as well share your life too (but maybe not on Github)! Just as with code, I'm sure we've all got tons to learn from each other about how to rock out this life thing.

And, of course, we can learn a bit from each other about what not to do as well.

Command Line Basics
A new way to use your computer!

Usually, when I'm navigating through folders and files on my computer, I click on a whole bunch of stuff. It takes a lot of time, finding the right folder with my eyes and then clicking on it. Click, click, click. Geez.

Turns out, there's a better way to communicate with your Macbook! Enter the command line.

Here are some cool commands you can use to get things done with your computer.

Identify your location

pwd - print working directory

Navigate between directories

cd - change directory
pushd - Save current directory, then go to another directory.
popd - Take the last directory pushed and "pop" it off (take me back there).

Move, rename, and copy items

mv - Move a file or directory
cp - Copy a file or directory

Create new and delete existing directories and files

mkdir - Make directory
rmdir - Remove directory
touch - Make a new empty file
rm - Remove a file

Open, view, or stream a file

open - Open a file
less - View a file
cat - Stream a file (display the whole file on the screen with no paging or stopping)

Find and look inside files

find - Find files
grep - Find things inside files

Get help

man - Read a command’s manual page
apropo - Look through all the help files to find potentially relevant help

That's a little bit about how you can use the Command Line! Check out Zed Shaw’s Learn Code The Hard Way for an excellent crash course.

Fixed Mindset Vs. Growth Mindset
Give Yourself Permission to Grow.

Be careful how you are talking to yourself because you are listening.
- Lisa M. Hayes

The quote above, spotted in a right-column ad gracing the border of the article this blog-post is about, rang true to me as I reflected upon Carol Dweck’s Fixed Mindset vs Growth Mindset.

Here’s the difference between the two mindsets:

Fixed Mindset

  • "Learning how to use Git will be tough. I don’t wanna."
  • "Arrays are kind of confusing. Fuck arrays."
  • "Spending more than 20 minutes on this method might prove that I’m a moron."
  • "Use Version Control, you say? Sounds like you just said that you hate me."
  • "Oh, you built this? I guess I suck."

Growth Mindset

  • "Learning how to use Git will be tough. Bring it on."
  • "Arrays are kind of confusing. Time to buckle down."
  • "The 3 hours I’ve spent on this method have served as important learning time. I’m heading in the right direction!"
  • "Use Version Control, you say? Makes sense. Sounds good."
  • "Oh, you built this? Can’t wait to tackle a big project of my own!"

Keep that positive self-talk coming! Don’t forget that someone is listening.

Personal Thinking Styles
Realist, Experimenter, Contemplator, or Theorist?

Learning preferences are pretty sweet, right? Thinking styles are also pretty sweet.

Professor Anthony Gregorc’s questionnaire on The Learning Web checks your Personal Thinking Style. Gregorc divided these into four groups:

Concrete Sequential Thinkers (CS) tend to be based in reality. They process information in an ordered, sequential, linear way.
Concrete Random Thinkers (CR) are experimenters.
Abstract Random Thinkers (AR) organize information through reflection, and thrive in unstructured, people-oriented environments.
Abstract Sequential Thinkers (AS) love the world of theory and abstract thought.

I scored equally high as both a Concrete and Abstract Random thinker. Here’s my scene according to a Boise State University PDF:

Concrete random thinkers are experimenters. says DePorter: "Like concrete sequentials, they’re based in reality, but are willing to take more of a trial-and-error approach. Because of this, they often make the intuitive leaps necessary for true creative thought. They have a strong need to find alternatives and do things in their own way." If you’re a CR, use your divergent thinking ability. Believe that it’s good to see things from more than one viewpoint. Put yourself in a position to solve problems. But give yourself deadlines. Accept your need for change. Try and work with people who value divergent thinking.

Abstract random thinkers organize information through reflection, and thrive in unstructured, people-oriented environments. Says DePorter: "The ‘real‘ world for abstract random learners is the world of feelings and emotions. The AR’s mind absorbs ideas, information and impressions and organizes them through reflection. They remember best if information is personalized. They feel constricted when they’re subjected to a very structured environment." If you’re an AR, use your natural ability to work with others. Recognize how strongly emotions influence your concentration. Build on your strength of learning by association. Look at the big picture first. Be careful to allow enough time to finish the job. Remind yourself to do things through plenty of visual clues, such as colored stickers pasted up where you’ll see them.

Emotional Intellegence
Search Inside Yoself.

So, I’m starting an intensive, 18-week Web Development Bootcamp in two weeks time! The excitement is figuratively killing me.

To ensure students are culturally on the same page before the first day of school, we’ve been assigned some self-initiation. This includes a Dev Bootcamp original vid on Emotional Intelligence (EQ).

Inspired by above funny guy Chade-Meng Tan’s project Search Inside Yourself, EQ is a trainable competency consisting of awareness, empathy, and intuition.

Awareness is the ability to catch an emotion as a bodily response early, slow it down, and watch it happen. You can then choose how to best handle your emotion; you own your emotion instead of your emotion owning you.

Empathy is the awareness of emotions in others as they arise. This includes knowing the difference between an ask and a need, or searching beyond words to understand the needs of others.

Intuition is a bodily sensation of the right path in the moment. It is the ability to listen to your body and not just your cognitive brain.

Interested in learning more about EQ? Watch Dev Bootcamp’s vid or cozy up to Chade-Meng Tan’s Search Inside Yourself.

Becoming a Feedback Ninja
Developing With Your Community :D

Because the folks at Dev Bootcamp believe that feedback is central to learning, their next video (created by DBC Head of Admissions Karim Bishay) outlines the four distinct skills required to become a feedback ninja: soliciting, feedback, receiving feedback, giving feedback, and integrating feedback.

Soliciting feedback requires knowing the difference between a request and a demand. When asking to give or receive feedback, make sure that the other person can say no without emotional punishment.

Receiving Recieving feedback calls for emotional control. Bishay shows us the Compass of Shame, a visual aid that summarizes different ways we respond to difficult feelings.

When you receive feedback and notice yourself coping with shame via one of the above strategies, allow yourself to become aware of that impulse and then chose not to completely give in to it.

Giving feedback must be actionable (something they can change), specific (pertaining to one particular action), and kind (honest about the impact their action had on you instead of projecting or judging).

Integrating feedback involves taking from feedback what is useful to you instead of accepting all feedback as universal truth.

That’s a little bit about feedback! Watch Dev Bootcamp’s vid to learn more.
