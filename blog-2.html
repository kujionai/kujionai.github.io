Ruby Classes
Greeting the wildlife with ordered expectations.

Not long after moving a stone's throw South of Brooklyn's Prospect Park about six months ago, I fell in love with the neighborhood's WILDLIFE. We're talking squirrels. Sparrows. And to my purest delight, we've got cats on cats on cats EVERYWHERE.

To Ruby-document some of my favorite feline neighbors, I've created the following class called Neighborhood_Cat. 

class Neighborhood_Cat
  def initialize(location, favorite_activity)
    @location = location
    @favorite_activity = favorite_activity
  end
  def hang_out
    puts "My favorite hangout spot is the #{@location}! Next time we meet, I'll definitely #{@favorite_activity}."
  end
  def meow
    puts "Meow! Meow!"
  end
end

Classes are perfect for grouping behaviors (methods) into convenient bundles that allow you to effortlessly create many objects that behave in basically the same way. By creating the class Neighborhood_Cat, I can now create objects that are instances of this class - different neightborhood cats that all meow and hang_out (these are the methods bundled in the class), but have different locations and favorite_activitys (these are the arguments passed into the initialize method).

Each Neighborhood cat has a favorite location and an expected favorite_activity, so I'll make those the arguments for Neighborhood_Cat's initialize method (initialize gets called each time we create a new instance of a class).

brown_cat = Neighborhood_Cat.new("corner deli checkout counter", "stay asleep while you try to get my attention")
brown_cat.hang_out
brown_cat.meow

black_cat = Neighborhood_Cat.new("overgrown lot next to the Catholic School", "run away and then look back at you like I want you to chase me")
black_cat.hang_out
black_cat.meow

grey_cat = Neighborhood_Cat.new("new duplex close to the subway stop", "butt my head against your hand")
grey_cat.hang_out
grey_cat.meow

Sweet! We created new instances of our Neighborhood_cats class and called the methods on each instance. Here's what we'll get when we run this in our terminal:

=> My favorite hangout spot is the corner deli checkout counter! Next time we meet, I'll definitely stay asleep while you try to get my attention.
=> Meow! Meow!
=> My favorite hangout spot is the overgrown lot next to the Catholic School! Next time we meet, I'll definitely run away and then look back at you like I want you to chase me.
=> Meow! Meow!
=> My favorite hangout spot is the new duplex close to the subway stop! Next time we meet, I'll definitely butt my head against your hand.
=> Meow! Meow!

Awesome! Keeping all those cats straight is pretty simple with a class. Don't forget to greet your furry neighbors!

Lighting the Fire
Seven Women of Color Opening The Doors to Web Development.

"I would tell her, never apologize for the space you occupy in the world. Make your strides long, wide and sure."
- Chimamanda Ngozi Adichie

So, I must say that I've been a black woman for quite a while now. I'm doing my best to shove my foot into the door of computer programming.

Let me tell you, that even with access to Dev Bootcamp's infinitely supportive community, the gateway to the Code-World feels hefty sometimes. I believe I've developed a mid-sized imposter-syndrome-shaped, eggplant-esque bruise on both pinky toes.

I'm certain, however, that this door is beginning to open for women and People of Color due to the amazing work of folks like the seven bad-ass hackers (listed in no particular order) you're about to read about. These ladies are commited to spreading the tech love around, and they are changing the face of computer programming.

Saron Yitbarek

Saron Yitbarek runs the weekly CodeNewbie TwitterChat with fellow Web Dev Carlos Lazo. This supportive community of programmers and people learning to code gathers every Wednesday at 9PM EST to connect through resources, stories, and support.

QUOTE "Take every opportunity you can, especially if it scares you."

Kathryn Finney

Kathryn Finney is the founder of DigitalUndivided, an organization that produces programs and events aimed at increasing the number of women of color in the tech industry.

QUOTE "Diversity in tech is important for everyone ... Be on the cutting edge, go where people aren’t looking. You’re going to have a very difficult time if you’re looking in the same pool as everyone else."

Amanda Spann

Inspired by Black Girls Code in San Francisco, Amanda Spann founded Black Girls Hack with three others. Black Girls Hack is the first non-profit hackathon series specifically targeting Women of Color.

QUOTE "We’re all African American, and we all have tech startups. One thing we all struggle with is feeling like we’re the only ones ... there are so few blacks in technology."

Felicia O’Garro

Felicia O’Garro decided to teach herself how to code while working full-time. She and her husband founded Code Crew, a flourishing learning space that has already taught thousands how to code in a year and a half.

QUOTE "When we found out that coding is something that you can teach yourself, we were overly excited."

Courtney Ervin

Courtney Ervin is a teacher at Dev Bootcamp’s Web Development Immersive. Because of her self-taught programming background, she loves having the opportunity to create supportive and challenging environments for new programmers.

QUOTE "Life goal: I want to be the @NICKIMINAJ of software engineering."

Kimberly Bryant

Kimberly Bryant launched Black Girls Code to empower and mentor young women of color by highlighting the achievement of women in tech. Since 2011, this non-profit has taught over two thousand girls ages 7-17 about computer programming and digital technology.

QUOTE "When bringing a new idea to life, be sure you start the journey with integrity. It’s equally if not more important than the idea itself."

Christina Lewis Halpern

An idealist based in New York City, Lewis Halpern prepares black and Latino teenage boys for awesome tech careers through her non-profit All Star Code.

QUOTE "I thought, If my father were a young man today, where would he want to go? He wouldn't want to go to law school. He'd want to be in Silicon Valley. So I thought, well, are there any black people in the space? No. Has it already been diversified? No."

So, that's my list! Be sure to follow these game-changers on twitter, and be proud to be you.

Ruby's Map Method
An Enumerable for the well-adapted glutton.

Let's say (with inspiration from a most clever Dev Bootcamp challenge) we've got some bagels, and want another bagel. And, of course, we wanna code about it.

more_bagels = ["I've" "got" 2 "bagels" "!" "Now" "I" "want" 3 "."]

Great! Awesome start. The array more_bagels Tells the whole wanting-another-bagel story. UNLESS, of course, I get that extra bagel, in which case it's only natural to want one more.

But, how might I freely adjust my more_bagels array to properly emulate my bottomless gluttony? It's time to bust out Ruby's #map method!

But why #map? Well, I want to change the numbers in my array based on the number of bagels I have, so I'll create a method that accepts both an array and a number as arguments. Unlike #each, which just does something to each element without storing the results anywhere, #map creates a new array with the results of whatever action you're performing. It's the same as the #collect method, so you can think of #map as 'collecting' the results of an action into a brand new array.

In my case, I'm executing some delicious addition unto all of the numbers in my array.

def bagels_on_bagels(array, num)
  array.map { |obj|
    if obj.is_a?Fixnum
      obj + num
    else
      obj
  end }
end

Cool! So, I'm letting that #map method have at it on my array argument, finding those Fixnums and adding some nums to them. Time to call my new method:

bagels_on_bagels(more_bagels, 7)

Alright! let's see what all this method madness returns.

=> ["I've" "got" 9 "bagels" "!" "Now" "I" "want" 10 "."]

Perfect! One more bagel, no matter how many I've already hoarded inside of my face. So, #map created a new array for me with the results of performing my method on each element! The #map sure is great for returning an ever-adjusting desire for more.

Wanna fill up on #map, #select and other life-changing Enumerable Methods? Indulge youself with Erik Trauman's lovely blog-post.
